# 0. DOCUMENT ROLE

This document defines the **official frontend build order**
for the Social Media Web App.

It answers:

- what must be built
- in what order
- why that order exists
- when a stage is considered complete

This document enforces **product reality + system architecture alignment**.

---

# 1. CORE PRINCIPLE — CONTENT LOOP FIRST

This roadmap is built around one fundamental truth:

User → Create → Feed → Interact → Repeat

Without Create Post:

- Feed becomes read-only
- Interaction becomes artificial
- Product becomes fake

Therefore:

> Create Post is part of the **core system loop**
> and MUST be implemented early.
